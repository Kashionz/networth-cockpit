## 1. 產品願景

### 1.1 一句話定位

> 把分散在各處的資產彙整在一個地方,看見配置現況與淨值成長軌跡,並透過規則引擎與 AI 提供貼著個人風險屬性的調整方向(教育性質,非投資建議)。

### 1.2 願景敘事

多數台灣上班族的財務狀況分布在多個系統:銀行 App 看活存、券商 App 看股票、信用卡帳單看消費、Excel 試算表算總帳。每個系統都只看到一塊拼圖,沒有人能在一個地方回答這個問題:

> 「我這個月真的有變更有錢嗎?如果有,變多少?該怎麼辦?」

本產品要回答的就是這個問題,並進一步幫使用者:

- 看到淨值隨時間成長的軌跡
- 知道目前資產配置偏離理想多少
- 在「亂花錢」、「儲蓄不足」、「投資加碼時機不對」時收到中性的提示
- 透過 AI 解讀層,把冷冰冰的數字翻成個人化的白話建議

### 1.3 不做什麼

明確列出**刻意不做**的事,避免 scope creep:

- 不做具體買賣標的建議(法規線)
- 不做高頻交易、量化策略、技術分析
- 不做信用卡循環餘額追蹤(目標用戶不會循環)
- 不做房貸 / 信貸 / 海外資產(目標用戶不會持有)
- 不做逐筆飲料便當的詳細記帳(摩擦過高,見第 4 章)
- 不做社群、不做炫富、不做攀比功能
- 不做投顧執照下的客製化資產管理

## 2. 目標用戶

### 2.1 主要用戶 Persona

**「自律但時間有限的上班族投資人」**

| 維度 | 描述 |
|---|---|
| 年齡 | 28–42 歲 |
| 收入 | 月薪 6–15 萬 |
| 投資經驗 | 1–10 年,以 ETF 與台股為主 |
| 財務狀態 | 信用卡全額繳清、有緊急金、月儲蓄率 15–35% |
| 心態 | 想做好理財但不想變成全職分析師 |
| 痛點 | 資料分散、無法看到全貌、不知道下一步該做什麼 |
| 工具現況 | Excel + 券商 App + Moneybook,各自為政 |

### 2.2 次要用戶

- **剛開始投資的職場新鮮人**(25–28 歲):還在建立財務習慣,需要更強的引導與教育
- **接近退休的中高齡投資人**(45–55 歲):資產較多元,需要更細的配置分析

### 2.3 明確排除的用戶

- 高槓桿玩家、當沖客、選擇權交易者
- 高負債族群(需要的是債務管理工具,不是這個)
- 高資產客戶(他們有專屬投顧)
- 完全不投資、只想記帳的人(市面上有更好的純記帳工具)

## 3. 核心使用情境

### 3.1 主要 user stories

**US-1 / 月度淨值檢視**
> 作為一個有定期投資習慣的上班族,我希望每個月底花 5 分鐘就能看到「這個月淨值變多少」、「儲蓄率多少」、「投資配置健不健康」,而不需要打開五個 App。

**US-2 / 配置偏離提醒**
> 作為一個有設定目標配置(例如股 70% / 債 20% / 現金 10%)的投資人,我希望在偏離超過某個門檻時收到提醒,知道「該不該 rebalance」與「補哪一塊」。

**US-3 / 預算控制**
> 作為一個容易月底發現錢花光的人,我希望能設定本月「生活」與「彈性」預算上限,並在用量達 80% 時收到提示,而不是月底事後反省。

**US-4 / 信用卡帳單一鍵記帳**
> 作為一個刷卡比例 > 80% 的使用者,我希望每月只需匯入一份信用卡帳單,系統就自動分類完成全月記帳,不需要逐筆輸入。

**US-5 / 教育型 AI 解讀**
> 作為一個看不懂專業術語的投資人,我希望系統能用白話跟我解釋「為什麼這個月配置變這樣」、「有什麼選擇可以做」,並明確標示這不是投資建議。

### 3.2 完整 journey (月度)

```
月初(1 號)
  ├─ 收到推播:「本月預算已分配,生活 15,000 / 彈性 7,000」
  └─ 月初 snapshot 自動生成

月中(刷卡時)
  ├─ 不需要做任何事(系統不要求逐筆記帳)
  └─ 預算用量達 80% 時收到提示推播

月底(信用卡結帳日 + 1 天)
  ├─ 推播:「OO 卡帳單已出,點此匯入」
  ├─ 上傳 CSV → 系統自動分類 → 使用者審核 5 分鐘
  └─ 月度報表自動生成

月底(最後一天)
  ├─ 月度淨值 snapshot
  ├─ 健康指標檢查 → 觸發警示(若有)
  └─ AI 月報生成(Phase 1)
```

## 4. 產品原則與記帳哲學

> **本章是整份 PRD 的靈魂。所有功能取捨都應回到這裡檢驗。**

### 4.1 為什麼記帳工具會失敗

所有手動記帳工具最終都會遇到同一個衰退曲線:第一週熱情,第三個月放棄。原因不是使用者懶,是**動機與摩擦的不對等**:

| 時間點 | 動機 | 摩擦 |
|---|---|---|
| 記帳當下 | 0(剛花錢的人不想再想到錢) | 中(打開 app、選分類、輸金額) |
| 看報表時 | 高 | 0 |

每筆交易要在「動機 = 0」的時刻付出「摩擦 > 0」的成本,換取要等 30 天後才兌現的報酬。這是個 broken loop。

本產品的核心設計目標是**徹底打破這個 loop**。

### 4.2 五大產品原則

**原則一:摩擦優先 (Friction First)**

每一個要求使用者主動輸入的欄位,都要回答「不問可以嗎?」如果答案是可以,就不問。投資資產自動拉行情、信用卡靠帳單匯入、收入靠每月一筆設定 — 全部目的是把每月互動時間壓在 10 分鐘以內。

**原則二:抓大放小**

只追蹤重複性、高金額、低頻率的支出。不要求使用者記每杯飲料、每個便當。系統的價值不在「記得每筆錢花去哪」,而在「看到結構與軌跡」。

**原則三:向前看,不向後評判**

預算制(向前看 + 規劃)優於記帳制(向後看 + 評斷)。前者讓使用者有控制感,後者讓使用者愧疚。文案、視覺、互動都遵循這個原則。

**原則四:中性,不羞辱**

不用紅字、不用驚嘆號、不用警告音、不做 Duolingo 式連勝紀錄。「彈性預算用完囉」不是「⚠️ 超支!」。財務工具不該帶情緒包袱。

**原則五:儲蓄率 > 配置優化**

長期淨值成長,儲蓄率的影響遠大於資產配置。系統的首頁第一個數字不是「總資產」,是「本月儲蓄率」。

### 4.3 記帳哲學:三層協作

**層 1:資產配置層 — 完全自動**

股票、ETF、現金餘額靠 API 自動更新。使用者唯一要做的是「我這個月又投入了 N 萬」這種偶爾的手動更新。**這層的記帳成本應該是 0**。

**層 2:現金流層 — 抓大放小 + 半自動**

只追蹤這幾類:

| 類別 | 追蹤方式 |
|---|---|
| 房租 / 月費 | 設定一次,每月自動 |
| 訂閱(Netflix、雲端、健身房) | 設定一次,每月自動 |
| 保險 | 設定一次,每年提醒 |
| 信用卡帳單 | 月結日後匯入,一次完成全月記帳 |
| 大筆採購(>3,000) | 手動,但**只有這類**要手動 |

**層 3:預算制風味 — 月度信封**

不讓使用者事後評判,而是事前承諾:

```
本月預算分配範例:
  固定支出:        18,000  [自動扣除]
  生活預算:        15,000  [信用卡刷,扣完進警示]
  彈性預算:         7,000  [信用卡刷,扣完進警示]
  儲蓄目標:        20,000  [優先撥到投資帳戶]
                ─────────
  總月可支配:      60,000
```

這個 framing 把金錢決策的時刻從「事後反省」移到「事前規劃 + 當下提示」,這才是真正改變行為的關鍵。

### 4.4 不做的功能(故意的)

| 功能 | 為什麼不做 |
|---|---|
| 每筆消費的詳細多級分類學 | 摩擦過高,且無 actionable 意義 |
| 拍發票辨識單筆消費 | 摩擦過高、邊際效益低 |
| 每日記帳推播提醒 | 教使用者煩躁的最快方法 |
| 連續記帳天數 / 連勝紀錄 | 帶情緒包袱、與「降低摩擦」原則衝突 |
| 全網最便宜商品比價 | 與「不評判」原則衝突 |
| 紅字超支警告 | 與「中性」原則衝突 |

## 6. 詳細功能規格

### 6.1 M1 — 資產彙整

**6.1.1 支援的資產類別**

| 類別 | 估值來源 | 估值頻率 | MVP |
|---|---|---|---|
| 活存 / 定存 | 手動輸入餘額 | 使用者更新 | ✅ |
| 貨幣基金 | 手動輸入 | 使用者更新 | ✅ |
| 台股個股 | 證交所 OpenAPI | 每日收盤後 | ✅ |
| 台股 ETF | 證交所 OpenAPI | 每日收盤後 | ✅ |
| 債券 ETF(台灣掛牌) | 證交所 / 櫃買 | 每日 | ✅ |
| 加密貨幣 | Coingecko | 每日(可選) | Phase 1 |
| 勞退自提 | 手動 | 季更新 | Phase 1 |
| 實物資產(車、金) | 手動 | 半年 | Phase 1 |

**6.1.2 互動流程**

新增資產 → 選類別 → 選/輸代號 → 輸入持有數量 + 成本 → 完成

編輯持有 → 顯示目前持有數 → 加減倉操作 → 自動記錄交易紀錄(可選關閉)

**6.1.3 自動估值機制**

```
每日 21:00(台股收盤後 1.5 小時)排程任務:
  1. 拉取所有 user 持有資產的代號
  2. 從證交所 OpenAPI 抓當日收盤價
  3. 寫入 prices_daily 表
  4. 觸發 user 投組重新計算
  5. 生成當日 portfolio_snapshot
```

### 6.2 M2 — 現金流追蹤

**6.2.1 支援的支付方式**

| 支付方式 | 處理邏輯 |
|---|---|
| 信用卡 | 立即記消費 + 累計到該卡本期應繳 |
| 現金 | 立即記消費 + 減少對應現金資產 |
| 銀行扣款 / 轉帳 | 立即記消費 + 減少對應銀行存款 |

**6.2.2 信用卡帳單匯入流程(MVP 核心功能)**

```
1. 使用者從發卡行下載帳單 CSV / PDF
2. 系統介面拖拉上傳
3. 解析引擎:
   - Phase 0:純規則(關鍵字匹配 + 上次規則記憶)
   - Phase 1:LLM 處理沒見過的商家
4. 顯示解析結果列表,標示「已自動分類 N 筆」「待審核 M 筆」
5. 使用者審核新商家 → 系統記住分類規則
6. 確認 → 全部寫入 transactions 表 + 凍結為 card_statement
```

**6.2.3 訂閱服務管理**

```
新增訂閱 → 選類型(影音/雲端/SaaS) → 輸入金額 + 扣款日 + 信用卡 → 完成
```

每月扣款日自動產生 transaction,使用者不需介入。

**6.2.4 大額手動記錄**

> 5,000 元以上的非信用卡消費(現金、轉帳)需要手動記錄。低於門檻的小額現金消費**不要求**記錄(原則:摩擦優先、抓大放小)。

### 6.3 M3 — 預算管理

**6.3.1 預算類別(預設三類,可自訂)**

| 類別 | 涵蓋範圍 | 預設行為 |
|---|---|---|
| 固定支出 | 房租、訂閱、保險、月費 | 月初預扣 |
| 生活預算 | 餐飲、交通、日用、雜支 | 信用卡刷扣 |
| 彈性預算 | 購物、娛樂、出遊 | 信用卡刷扣 |

**6.3.2 預算設定流程**

```
首次設定:
  ├─ 系統建議:基於上月信用卡帳單拆分(若有)
  ├─ 或:詢問月薪 → 套用 50/30/20 模板
  └─ 使用者調整 → 儲存

月度滾動:
  ├─ 預設沿用上月分配
  ├─ 月初 1 號自動激活
  └─ 月底結算後生成達成率
```

**6.3.3 預算警示機制**

| 用量 | 行為 |
|---|---|
| < 80% | 不主動通知,儀表板顯示進度 |
| 80–99% | 推播一次:「彈性預算還剩 X 元,N 天」 |
| ≥ 100% | 推播一次(中性文案),並在儀表板標示 |
| 超支 > 20% | 月底提示是否調整下月分配 |

**6.3.4 滾動規則**

預算類別可設定 `rollover` 屬性:當月未用完是否進入下月。預設彈性預算可滾、生活預算不滾。

### 6.4 M4 — 風險屬性評估

**6.4.1 問卷設計**

8–10 題,涵蓋:

- 年齡與投資年限
- 主要投資目的(退休、購房、子女教育、財務自由)
- 流動性需求(短期是否會動用)
- 虧損忍受度(假設下跌 30% 你會...)
- 投資知識自評
- 收入穩定性

**6.4.2 五級分類與目標配置**

| 等級 | 標籤 | 股票 | 債券 | 現金 |
|---|---|---|---|---|
| L1 | 保守 | 20% | 50% | 30% |
| L2 | 穩健保守 | 40% | 40% | 20% |
| L3 | 平衡 | 60% | 30% | 10% |
| L4 | 穩健成長 | 75% | 20% | 5% |
| L5 | 積極 | 90% | 5% | 5% |

> 上述配置為**預設參考**,使用者可在問卷後手動調整目標配置。

**6.4.3 重新評估**

每 12 個月或重大生活事件(換工作、結婚、生子)後提示重新填寫。

### 6.5 M5 — 配置分析

**6.5.1 主要視圖**

- **資產配置餅圖**:現況 vs 目標,並排顯示
- **偏離度條**:各類別偏離 ±%
- **個股集中度**:Top 5 持股佔比,>20% 觸發提示
- **相關性矩陣**:Phase 1 功能,讓使用者看到持股相關性是否過高

**6.5.2 Rebalance 提醒**

| 觸發條件 | 行為 |
|---|---|
| 任一類別偏離 > 5% 且持續 7 天 | 推播一次中性提示 |
| 偏離 > 10% | 推播一次 + 儀表板高亮 |
| 偏離 > 15% | 月度報告中專章說明 |

**6.5.3 補倉建議(教育型)**

當使用者輸入「下月可投入 N 萬」時,系統根據偏離度建議**比例**(不指定標的):

> 建議分配方向:股票類別 60%,債券類別 30%,現金 10%。實際標的請依個人偏好選擇。

### 6.6 M6 — 健康警示

**6.6.1 警示規則樹(規則層 L1)**

```
判斷順序(從上到下,首條滿足即觸發):

1. 本期信用卡應繳 > 可動用現金 ?
   → 緊急警示(紅色,但不用驚嘆號):
     「本月帳單可能繳不出來,建議立即檢視」

2. 緊急預備金月數 < 3 ?
   → 黃色提示:
     「流動性偏低,建議先補緊急金再加碼投資」

3. 儲蓄率 < 10% ?
   → 藍色教育型提示:
     「長期累積需要先擴大儲蓄率,可考慮重新分配預算」

4. 投資配置偏離 > 10% ?
   → 灰色資訊型提示:
     「資產配置已偏離目標,可考慮 rebalance」

5. 全部健康 → 顯示綠色 OK 卡片
```

**6.6.2 文案設計原則**

- 不使用「警告」、「危險」、「錯誤」等情緒詞
- 用「建議」、「可考慮」、「值得檢視」等中性詞
- 不使用紅字以外的視覺刺激(不彈窗、不音效、不震動)
- 每條警示必附「為什麼」一句說明,且不超過兩行

### 6.7 M7 — 智慧引擎(三層架構)

**6.7.1 Layer 1 — 規則層(deterministic)**

純 if/else 與閾值判斷,完全可預測、可測試。內容:

- 健康警示樹(見 6.6.1)
- 目標配置區間(見 6.4.2)
- Rebalance 觸發閾值(見 6.5.2)
- 預算警示閾值(見 6.3.3)

實作:TypeScript 純函式,跑在 Supabase Edge Function。

**6.7.2 Layer 2 — 量化分析層(numerical)**

數值計算與歷史回測。內容:

- 投組績效計算(報酬率、波動率、Sharpe、最大回撤)
- 與基準比較(0050、SPY、60/40 portfolio)
- 簡單情境壓力測試(市場 -20%、升息 100bps)

實作:Python + pandas/numpy/scipy,跑在 FastAPI 服務(內部呼叫,不對 user 開放)。

**6.7.3 Layer 3 — AI 解讀層(LLM)**

把 L1 + L2 的數字翻譯成個人化白話說明。**嚴格約束**:

- 不准給具體買賣指令
- 不准預測短期漲跌
- 數字一律從 L1/L2 注入,LLM 只負責改寫
- 所有輸出附免責聲明
- 中性、不批判、不情緒化

模型選擇:GPT-4o 或 Claude Sonnet。Prompt template 在第 10 章詳述。

**6.7.4 三層協作範例**

```
情境:使用者本月儲蓄率 8%,股票部位偏離目標 +12%

L1 規則層輸出:
  flags: ["low_savings_rate", "allocation_drift_high"]
  thresholds: {savings: 8, target: 10, drift: 12, threshold: 10}

L2 量化層輸出:
  current_allocation: {stock: 72, bond: 23, cash: 5}
  target_allocation: {stock: 60, bond: 30, cash: 10}
  ytd_return: 8.4%
  benchmark_return: 7.1%

L3 AI 解讀層輸出:
  「本月儲蓄率 8%,略低於 10% 的健康下限。可考慮先檢視
   本月預算分配,看看哪一類有調整空間。
   
   投資配置上,股票部位目前佔 72%,高於設定的 60% 目標約
   12%,主要因為近期市場上漲。當有新資金投入時,可優先補
   債券或現金部位來回到目標配置。
   
   ※ 以上為教育性質的資訊,不構成個別投資建議。」
```

### 6.8 M8 — 報表與軌跡

**6.8.1 月度報告**

每月 1 號自動生成上月報告:

- 淨值變化金額 + %
- 儲蓄率達成
- 預算達成率(三類)
- 投資組合績效
- 健康警示總覽
- AI 解讀(Phase 1)

**6.8.2 淨值時間線**

堆疊面積圖:正向是各類資產、負向是信用卡未繳款,中線是淨值。

**6.8.3 里程碑系統**

中性、無遊戲化壓力的里程碑:

- 淨值首次破 50 萬 / 100 萬 / 200 萬 ...
- 緊急金達 6 個月支出
- 連續 N 個月儲蓄率 ≥ 20%
- 投資資產 break even

每個里程碑只在達成當下顯示一次,不做連勝、不做排行榜。

### 6.9 M9 — 設定與安全

**6.9.1 隱私模式**

一鍵切換「金額隱藏」模式,所有金額顯示為 ¥¥¥,只顯示比例與趨勢。給使用者在公開場合查看的安全感。

**6.9.2 資料匯出**

完整 CSV / JSON 匯出,使用者隨時可下載自己所有資料。

**6.9.3 帳號刪除**

提供完整帳號刪除流程,30 天緩衝期後永久刪除。

## 11. 法規與合規

### 11.1 定位再確認

本產品定位為**個人財務管理工具(PFM)**,提供:

- 資料彙整與視覺化(資料服務)
- 一般性財務教育內容(資訊服務)
- 規則型健康提示(資訊服務)

**明確不提供**:

- 個別證券買賣建議
- 客製化投資組合管理
- 任何形式的代客操作

依此定位,不需要證券投資顧問執照。

### 11.2 必須的免責機制

- 註冊時需勾選「我了解本服務不提供投資建議」
- 每份 AI 解讀文末必附免責聲明
- 月度報告 footer 必附「本資訊僅供參考,不構成投資建議」
- 約莫每 30 天向使用者再次提示一次

### 11.3 個資保護

- 遵循台灣《個人資料保護法》
- 不收集身分證字號、不收集銀行帳號(只收集帳戶名稱與餘額)
- 信用卡只記末四碼,不存全卡號
- 資料傳輸 TLS、靜態加密
- 使用者可隨時匯出 / 刪除全部資料

### 11.4 LLM 隱私

- 送往 LLM 的 context 不包含可識別個資(姓名、卡號、帳戶號)
- 只送統計型數字(金額、百分比、類別代碼)
- 與 OpenAI / Anthropic 簽訂 zero data retention(若條款允許)

### 11.5 必須在上線前完成的法律審查

- 律師審查所有 AI 解讀模板
- 律師審查使用者條款與隱私政策
- 律師審查 marketing 文案(避免「投資建議」、「報酬保證」等用詞)
